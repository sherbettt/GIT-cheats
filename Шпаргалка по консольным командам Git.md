# Шпаргалка по консольным командам Git


## Общее

Git — система контроля версий (файлов). Что-то вроде возможности сохраняться в компьютерных играх (в Git эквивалент игрового сохранения — коммит). **Важно**: добавление файлов к «сохранению» двухступенчатое: сначала добавляем файл в индекс (`git add`), потом «сохраняем» (`git commit`).

Любой файл в директории существующего репозитория может находиться или не находиться под версионным контролем (отслеживаемые и неотслеживаемые).

Отслеживаемые файлы могут быть в 3-х состояниях: неизменённые, изменённые, проиндексированные (готовые к коммиту).

### Ключ к пониманию

Ключ к пониманию концепции git — знание о «трех деревьях»:

- Рабочая директория — файловая система проекта (те файлы, с которыми вы работаете).
- Индекс — список отслеживаемых git-ом файлов и директорий, промежуточное хранилище изменений (редактирование, удаление отслеживаемых файлов).
- Директория `.git/` — все данные контроля версий этого проекта (вся история разработки: коммиты, ветки, теги и пр.).

Коммит — «сохранение» (хранит набор изменений, сделанный в рабочей директории с момента предыдущего коммита). Коммит неизменен, его нельзя отредактировать.

У всех коммитов (кроме самого первого) есть один или более родительских коммитов, поскольку коммиты хранят изменения от предыдущих состояний.

### Простейший цикл работ

- Редактирование, добавление, удаление файлов (собственно, работа).
- Индексация/добавление файлов в индекс (указание для git какие изменения нужно будет закоммитить).
- Коммит (фиксация изменений).
- Возврат к шагу 1 или отход ко сну.

### Указатели

- `HEAD` — указатель на текущий коммит или на текущую ветку (то есть, в любом случае, на коммит). Указывает на родителя коммита, который будет создан следующим.
- `ORIG_HEAD` — указатель на коммит, с которого вы только что переместили `HEAD` (командой `git reset ...`, например).
- Ветка (`master`, `develop` etc.) — указатель на коммит. При добавлении коммита, указатель ветки перемещается с родительского коммита на новый.
- Теги — простые указатели на коммиты. Не перемещаются.



### Настройки

Перед началом работы нужно выполнить некоторые настройки:

```bash
git config --global user.name "Your Name"          # указать имя, которым будут подписаны коммиты
git config --global user.email "e@w.com"           # указать электропочту, которая будет в описании коммитера
git config --global core.autocrlf true             # (для Windows) включить преобразование окончаний строк из CRLF в LF
git config --global core.editor "code --wait"      # установить редактор по умолчанию (VS Code)
git config --global color.ui auto                  # включить цветной вывод в консоли
git config --global alias.co checkout              # создать алиас (сокращение) для команды checkout
git config --global alias.st status                 # создать алиас для status
git config --global alias.ci commit                 # создать алиас для commit
git config --global alias.br branch                 # создать алиас для branch
git config --global alias.hist "log --pretty=format:'%h %ad | %s%d [%an]' --graph --date=short" # свой формат лога

git config --local user.name "Кирилл Кораблин"     # локальная настройка (только для текущего репозитория)
git config --local user.email "k@runtel.ru"        # локальная настройка

git config user.name "Кирилл Кораблин"              # показать текущее значение настройки
git config --list                                   # показать все настройки
```


### Указание неотслеживаемых файлов

Файлы и директории, которые не нужно включать в репозиторий, указываются в файле `.gitignore`. Обычно это устанавливаемые зависимости (`node_modules/`, `bower_components/`), готовая сборка `build/` или `dist/` и подобные, создаваемые при установке или запуске. Каждый файл или директория указываются с новой строки, [возможно использование шаблонов](http://git-scm.com/book/ru/v2/%D0%9E%D1%81%D0%BD%D0%BE%D0%B2%D1%8B-Git-%D0%97%D0%B0%D0%BF%D0%B8%D1%81%D1%8C-%D0%B8%D0%B7%D0%BC%D0%B5%D0%BD%D0%B5%D0%BD%D0%B8%D0%B9-%D0%B2-%D1%80%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D0%B9#Игнорирование-файлов).

```bash
# Пример .gitignore
node_modules/
dist/
build/
*.log
.DS_Store
.env
.vscode/
```

### Длинный вывод в консоли: Vim

Вызов некоторых консольных команд приводит к необходимости очень длинного вывода в консоль (пример: вывод истории всех изменений в файле командой `git log -p fileName.txt`). При этом прямо в консоли запускается редактор [Vim](https://ru.wikipedia.org/wiki/Vim). Он работает в нескольких режимах, из которых Вас заинтересуют режим вставки (редактирование текста) и нормальный (командный) режим. Чтобы попасть из Vim обратно в консоль, нужно в командном режиме ввести <kbd>:q</kbd>. Переход в командный режим из любого другого: <kbd>Esc</kbd>.

Если нужно что-то написать, нажмите <kbd>i</kbd> — это переход в режим вставки текста. Если нужно сохранить изменения, перейдите в командный режим и наберите <kbd>:w</kbd>.

### Vim (некоторые команды)

```bash
# Нажатия кнопок
ESC     — переход в командный режим
i       — переход в режим редактирования текста
ZQ (зажат Shift, поочередное нажатие) — выход без сохранения
ZZ (зажат Shift, поочередное нажатие) — сохранить и выйти

# Ввод в командном режиме
:q!             — выйти без сохранения
:wq             — сохранить файл и выйти
:w filename.txt — сохранить файл как filename.txt

```


## Консольные команды

### Создать новый репозиторий

```bash
git init                      # создать новый проект в текущей директории
git init folder-name          # создать новый проект в указанной директории
```


### Клонирование репозитория

```bash
# клонировать удаленный репозиторий в одноименную директорию
git clone https://github.com/cyberspacedk/Git-commands.git    

# клонировать удаленный репозиторий в директорию «FolderName»
git clone https://github.com/cyberspacedk/Git-commands.git FolderName 

# клонировать репозиторий в текущую директорию
git clone https://github.com:nicothin/web-design.git .           

# клонировать только последний коммит (глубиной 1, для больших репозиториев)
git clone --depth 1 https://github.com/cyberspacedk/Git-commands.git

# клонировать конкретную ветку
git clone -b develop https://github.com/cyberspacedk/Git-commands.git
```


### Просмотр изменений

```bash
git status                      # показать состояние репозитория (отслеживаемые, изменённые, новые файлы и пр.)
git status -s                    # показать состояние в сокращенном формате
git diff                         # сравнить рабочую директорию и индекс (неотслеживаемые файлы ИГНОРИРУЮТСЯ)
git diff --color-words           # сравнить рабочую директорию и индекс, показать отличия в словах
git diff index.html              # сравнить файл из рабочей директории и индекс
git diff HEAD                    # сравнить рабочую директорию и коммит, на который указывает HEAD
git diff --staged                 # сравнить индекс и коммит с HEAD
git diff master feature           # посмотреть что сделано в ветке feature по сравнению с веткой master
git diff --name-only master feature # посмотреть что сделано в ветке feature, показать только имена файлов
git diff master...feature         # посмотреть что сделано в ветке feature с момента расхождения с master
git diff --stat                   # показать статистику изменений
git diff --cached                 # то же, что и --staged
git diff HEAD~2 HEAD              # сравнить два указанных коммита
git diff --word-diff              # показать изменения на уровне слов (более наглядно)
```


### Добавление изменений в индекс

```bash
git add .                         # добавить в индекс все новые, изменённые, удалённые файлы из текущей директории
git add text.txt                  # добавить в индекс указанный файл
git add -i                        # запустить интерактивную оболочку для добавления в индекс только выбранных файлов
git add -p                        # показать новые/изменённые файлы по очереди с вопросом об индексировании
git add -A                         # добавить все изменения во всем рабочем дереве (аналог git add .)
git add --all                      # то же, что и -A
git add -u                         # добавить изменения только отслеживаемых файлов
```


### Удаление изменений из индекса

```bash
git reset                          # убрать из индекса все добавленные в него изменения (в рабочей директории всё сохранится)
git reset readme.txt               # убрать из индекса изменения указанного файла
git reset HEAD readme.txt          # то же, что и выше (более явно)
git restore --staged readme.txt    # современный способ убрать файл из индекса (git >= 2.23)
```


### Отмена изменений

```bash
git checkout text.txt              # ОПАСНО: отменить изменения в файле, вернуть состояние файла из индекса
git restore text.txt                # современный способ отменить изменения в файле (git >= 2.23)
git reset --hard                    # ОПАСНО: отменить все изменения; вернуть состояние коммита HEAD
git clean -df                       # удалить неотслеживаемые файлы и директории
git clean -n                        # показать, что будет удалено (без реального удаления)
git checkout -- .                   # отменить все изменения в текущей директории
git checkout -f                      # принудительно отменить изменения и переключиться на ветку
```


### Коммиты

```bash
git commit -m "Name of commit"       # закоммитить проиндексированные изменения
git commit -a -m "Name of commit"    # проиндексировать отслеживаемые файлы (ТОЛЬКО отслеживаемые) и закоммитить
git commit -am "Name of commit"       # то же, что и выше (сокращенная форма)
git commit --amend -m "New message"   # исправить сообщение последнего коммита
git commit --amend --no-edit          # добавить изменения в последний коммит без изменения сообщения
git commit --dry-run                   # показать, что будет закоммичено, без реального коммита
git commit --allow-empty -m "Empty"    # создать пустой коммит
```


### Отмена коммитов и перемещение по истории

**Все команды, которые меняют историю (`reset`, `commit --amend`, `rebase`) можно выполнять ТОЛЬКО если коммиты еще не были отправлены в удалённый репозиторий.**

```bash
# Безопасная отмена (создает новый коммит)
git revert HEAD --no-edit               # отменить последний коммит новым коммитом
git revert b9533bb --no-edit            # отменить изменения коммита с указанным хешем
git revert HEAD~3..HEAD                  # отменить диапазон коммитов
git revert --abort                       # прервать конфликтный revert

# ОПАСНО: изменение истории (только для локальных коммитов!)
git commit --amend -m "Название"         # переделать последний коммит
git reset --hard @~                       # передвинуть HEAD (и ветку) на предыдущий коммит
git reset --hard 75e2d51                  # передвинуть HEAD на коммит с указанным хешем
git reset --soft @~                        # передвинуть HEAD на предыдущий коммит, но изменения оставить в индексе
git reset --soft @~2                       # то же, но на 2 коммита назад
git reset @~                                # передвинуть HEAD на предыдущий коммит, индекс сбросить, изменения оставить
git reset --keep @~                         # безопасный reset: не даст потерять незакоммиченные изменения
git reset --hard origin/master              # принудительно сделать локальную ветку такой же как на сервере
```


### Временно переключиться на другой коммит

```bash
git checkout b9533bb               # переключиться на коммит с указанным хешем (состояние "detached HEAD")
git checkout master                 # вернуться на ветку master
git checkout HEAD~2                 # переключиться на 2 коммита назад
```


### Переключиться на другой коммит и продолжить работу с него

```bash
git checkout -b new-branch 5589877   # создать ветку new-branch, начинающуюся с коммита 5589877
git switch -c new-branch 5589877      # современный аналог (git >= 2.23)
git checkout -B new-branch 5589877    # создать или принудительно переместить ветку
```


### Восстановление изменений

```bash
git checkout 5589877 index.html      # восстановить файл на момент коммита 5589877 (и добавить в индекс)
git restore -s 5589877 index.html    # современный аналог (git >= 2.23)
git restore index.html                # отменить изменения в файле (версия из индекса)
git restore --source=HEAD~2 index.html # восстановить файл из коммита 2 назад
```


### Копирование коммита (перенос коммитов)

```bash
git cherry-pick 5589877               # скопировать изменения из указанного коммита на активную ветку
git cherry-pick master~2..master       # скопировать 2 последних коммита из master
git cherry-pick -n 5589877              # скопировать изменения, но НЕ КОММИТИТЬ
git cherry-pick --abort                 # прервать конфликтный перенос
git cherry-pick --continue               # продолжить конфликтный перенос
git cherry-pick --skip                   # пропустить конфликтный коммит
```


### Удаление файла

```bash
git rm text.txt                        # удалить отслеживаемый неизменённый файл и проиндексировать
git rm -f text.txt                      # удалить отслеживаемый изменённый файл и проиндексировать
git rm -r log/                           # удалить всё содержимое директории log/ и проиндексировать
git rm ind*                               # удалить все файлы, начинающиеся на "ind"
git rm --cached readme.txt                # удалить из отслеживаемых, но файл оставить на месте
git rm -n text.txt                         # показать, что будет удалено (без реального удаления)
```


### Перемещение/переименование файлов

```bash
git mv text.txt test_new.txt           # переименовать файл и проиндексировать изменение
git mv readme_new.md folder/            # переместить файл в директорию folder/
git mv folder1/file.txt folder2/        # переместить файл из одной директории в другую
```


### История коммитов

Выход из длинного лога вывода: `q`.

```bash
git log                                 # показать все коммиты в активной ветке
git log -2                               # показать последние 2 коммита
git log -2 --stat                         # показать последние 2 коммита со статистикой
git log -p -22                             # показать последние 22 коммита с изменениями
git log --graph -10                         # показать последние 10 коммитов с ASCII-графом ветвления
git log --since=2.weeks                      # показать коммиты за последние 2 недели
git log --after '2023-06-30'                  # показать коммиты после указанной даты
git log --before '2023-07-01'                  # показать коммиты до указанной даты
git log --author="Kirill"                      # показать коммиты указанного автора
git log --grep fix                              # показать коммиты с "fix" в сообщении
git log --grep fix -i                            # регистронезависимый поиск
git log index.html                                # показать историю изменений файла
git log -p index.html                              # показать историю файла с изменениями
git log -G'myFunction' -p                           # показать коммиты, где менялась myFunction
git log --pretty=oneline                             # показать коммиты в одну строку
git log --pretty=format:"%h - %an, %ar : %s" -4      # форматированный вывод
git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short # мой любимый формат
git log master..branch_99                             # коммиты из branch_99, не влитые в master
git log branch_99..master                              # коммиты из master, не влитые в branch_99
git log --oneline                                       # очень краткий формат
git log --oneline --graph --all --decorate             # полная картина репозитория

# Разница между локальной и удаленной веткой
git log --oneline origin/main..main    # что есть у вас, но нет на удаленном
git log --oneline main..origin/main    # что есть на удаленном, но нет у вас

# Просмотр конкретного коммита
git show 60d6582                        # показать изменения из коммита
git show HEAD~                           # показать данные о предыдущем коммите
git show @~                               # аналогично
git show @~:index.html                     # показать контент файла на момент предыдущего коммита
git show :/"подвал"                         # показать самый новый коммит со словом "подвал" в описании

# Статистика
git shortlog -sn                           # показать количество коммитов по авторам
git shortlog -sn --all                       # по всем веткам
git log --numstat                             # показать статистику изменений (добавлено/удалено строк)
```


### Кто написал строку

```bash
git blame README.md                        # показать, кто и когда менял каждую строку файла
git blame README.md --date=short -L 5,8     # строки 5-8, только дата без времени
git blame -L 10,+20 README.md                # начиная с 10 строки, 20 строк
git blame -w -C -C -C README.md              # игнорировать пробелы, искать копирования
```

### Ветки

```bash
git branch                                 # показать список локальных веток
git branch -v                               # показать ветки и последний коммит в каждой
git branch -vv                               # показать ветки и отслеживаемые удаленные ветки
git branch -a                                 # показать все ветки (локальные и удаленные)
git branch -r                                  # показать только удаленные ветки
git branch new_branch                           # создать новую ветку на текущем коммите
git branch new_branch 5589877                    # создать ветку на указанном коммите
git branch -f master 5589877                      # переместить ветку master на указанный коммит
git checkout new_branch                            # перейти в ветку
git checkout -b new_branch                          # создать ветку и перейти в неё
git switch new_branch                                 # современный способ переключиться (git >= 2.23)
git switch -c new_branch                               # современный способ создать и переключиться
git checkout -B master 5589877                          # переместить ветку master и перейти в неё
git merge hotfix                                         # влить ветку hotfix в текущую
git merge hotfix -m "Горячая правка"                      # с указанием сообщения коммита слияния
git merge hotfix --log                                      # добавить сообщения вливаемых коммитов в описание
git merge hotfix --no-ff                                     # запретить fast-forward (всегда создавать коммит слияния)
git merge hotfix --squash                                     # влить изменения как один коммит (без истории ветки)
git branch -d hotfix                                           # удалить ветку hotfix (только если слита)
git branch -D hotfix                                            # принудительно удалить ветку (даже если не слита)
git branch --merged                                             # показать ветки, слитые с активной
git branch --no-merged                                          # показать ветки, не слитые с активной
git branch -m old_name new_name                                 # переименовать ветку
git branch -m new_name                                           # переименовать текущую ветку
git branch --unset-upstream                                      # убрать связь с удаленной веткой
git fetch origin                                                 # получить информацию об удаленных ветках
git checkout --track origin/github_branch                         # создать локальную ветку из удаленной и переключиться
```


### Теги

```bash
git tag                                 # посмотреть все теги
git tag -l "v1.*"                        # посмотреть теги, начинающиеся с v1.
git tag v1.0.0                            # создать легковесный тег на текущем коммите
git tag -a v1.0.1 -m "Релиз 1.0.1"        # создать аннотированный тег с сообщением
git tag -a v1.0.2 -m "Версия 1.0.2" abc1234 # тег на конкретный коммит
git tag -d v1.0.0                          # удалить локальный тег
git show v1.0.0                             # показать информацию о теге
git tag -n                                   # показать теги с сообщениями коммитов
git tag -n -l 'v1.*'                          # показать теги v1.* с сообщениями

# Работа с удаленными тегами
git push origin v1.0.0                        # отправить конкретный тег
git push origin --tags                         # отправить все локальные теги
git push origin --follow-tags                   # отправить только аннотированные теги
git push origin --delete v1.0.0                  # удалить тег на удаленном репозитории
git fetch --tags                                  # получить все теги с удаленного репозитория
git pull --tags                                    # получить изменения вместе с тегами
git ls-remote --tags origin                        # посмотреть теги на удаленном репозитории
```


### Временное сохранение изменений без коммита (Stash)

```bash

git stash drop                                  # удалить последнее сохранение
git stash drop stash@{2}                         # удалить конкретное сохранение
git stash clear                                  # удалить все сохранения

git checkout stash -- index.html                     # восстановить конкретный файл из stash
```

### Конфликт слияния

```bash

# Работа с конфликтами

git checkout --theirs index.html                 # оставить их версию
git checkout --merge index.html                   # показать конфликтную разметку для ручного редактирования
git checkout --conflict=diff3 --merge index.html  # показать конфликт с общим предком
git mergetool                                     # запустить визуальный инструмент слияния
git log --merge                                    # показать коммиты, вызвавшие конфликт
```

```bash
git rebase --continue                            # продолжить после разрешения конфликта
git rebase --skip                                 # пропустить конфликтный коммит

# Интерактивный rebase (ключевые команды внутри редактора)

# reword = использовать, но изменить сообщение
# edit = использовать, но остановиться для правки

```

### Ad-hoc / Полезные однострочники

```bash
# Удалить все локальные ветки, кроме master
git branch | grep -v "master" | xargs git branch -D

# Удалить все слитые ветки (кроме текущей и master)
git branch --merged | grep -v "\*\|master" | xargs -n 1 git branch -d

# Удалить все неотслеживаемые файлы (включая игнорируемые)
git clean -xfd

# Показать размер каждого файла в текущем коммите
git ls-tree -r -l HEAD | awk '{print $3, $4, $5}' | sort -k3 -rn | head -20

# Показать последний коммит по каждому файлу
git ls-tree -r -z --name-only HEAD | xargs -0 -n1 git --no-pager log -1 --pretty=format:"%h %cr %s" --

# Найти все коммиты, удалившие конкретную строку
git log -S"string_to_search" --source --all

# Показать статистику изменений автора за последний месяц
git log --author="Name" --since="1 month ago" --pretty=tformat: --numstat | awk '{ add += $1; subs += $2; loc += $1 - $2 } END { printf "added lines: %s, removed lines: %s, total lines: %s\n", add, subs, loc }'

# Восстановить удаленный файл из предыдущего коммита
git checkout $(git rev-list -n 1 HEAD -- "deleted/file.txt")^ -- "deleted/file.txt"

# Отменить все незакоммиченные изменения
git checkout -- .

# Показать файлы, измененные в последнем коммите
git diff-tree --no-commit-id --name-only -r HEAD

# Закоммитить с текущей датой (обмануть систему)
git commit --date="`date -R`" -m "message"

# Сжать репозиторий (после большой переписки истории)
git gc --aggressive --prune=now

# Посмотреть, что изменилось между двумя датами
git log --pretty=format:"%h %s" --after="2023-01-01" --before="2023-12-31"

# Переключиться на предыдущую ветку
git checkout -

# Показать все коммиты, изменившие конкретную функцию
git log -L :myFunction:path/to/file.js
```
